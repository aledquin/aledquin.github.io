Translation & blending pipeline

How the web app turns English EPUB blocks into progressive Spanish mixing and optional full-sentence Spanish. This page is static; open from the dev server at /docs/translation-pipeline.html.

High-level flow

flowchart TD
  A[EPUB bytes] --> B[extractEpub]
  B --> C[ContentBlock]
  D[Spanish EPUB] --> E[attachCompanion]
  C --> E
  E --> F[blocks plus plainEs]
  F --> G[Progressive blend]
  G --> H[Inline lang es spans]
  H --> I{Sentence mode?}
  I -->|Off or tap| J[Save blendedHtml]
  I -->|Replace| K{Selective?}
  K -->|No| L[sentenceLayer]
  K -->|Yes| M[selectiveSentenceBlend]
  L --> N[pr-sentence-mt seg]
  M --> N
  N --> J

Decision: which sentence path?

flowchart LR
  S[Settings] --> T{Sentence on?}
  T -->|No| P[Progressive only]
  T -->|Yes| U{Style}
  U -->|tap| P
  U -->|replace par or sent from start| V[sentenceLayer]
  U -->|sent after sightings| W[selective blend]
  U -->|par after sightings| V

Selective sentence logic (per lemma)

Only when Replace sentence and After lexicon sightings are on. Bundled Spanish is used for sentences that qualify; API is skipped when plainEs aligns. 

flowchart TD
  R[Scan book order] --> S[Split sentences]
  S --> T[Count lemma hits]
  T --> Q{Any lemma in sentence at least N times?}
  Q -->|No| B[Progressive sentence]
  Q -->|Yes| X[Bundled ES or API]
  B --> N[Join paragraph]
  X --> N

Fixture sample (no API in tests)

The repo ships reader-feature-sample.blocks.json (20 blocks, all with plainEs). Vitest asserts markup classes and Spanish content for every bundled scenario. Regenerate with: 

cd apps/web
npm run epub:feature-sample
